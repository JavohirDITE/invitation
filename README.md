# Пригласительное — Сергей и Юлия

Одностраничный адаптивный сайт-приглашение на свадьбу с анимацией конверта,
галереей, таймером, кнопкой карты и отдельным конструктором палитры.

## Структура

```
index.html              — само приглашение
demo.html               — конструктор палитры (для согласования цветов)
assets/
  css/style.css         — стили (палитра задаётся CSS-переменными)
  js/config.js          — НАСТРОЙКИ: дата, время, локация, список фото
  js/main.js            — логика: конверт, reveal, галерея, карта, таймер
  js/palette.js         — логика конструктора палитры
  js/demo-config.js     — настройка отправки в Telegram
photo/                  — 10 фотографий
decor/                  — референсы декора
```

## Что менять под себя

- **Дата / время / локация / фото** → `assets/js/config.js`
  - координаты площадки (`lat`, `lng`) или текстовый поиск (`query`)
- **Цвета** → переменные в начале `assets/css/style.css` (или подобрать в `demo.html`)
- **Отправка палитры в Telegram** → `assets/js/demo-config.js` (инструкция внутри файла)

## Кнопка карты

Определяет устройство:
- iPhone/iPad → Apple Maps
- Android → Google Maps (приложение)
- Десктоп → Google Maps в браузере

Координаты берутся из `config.js`.

## Конструктор палитры (demo.html)

Открывается рядом живой предпросмотр сайта. Можно:
- выбрать готовую гамму или каждый цвет вручную (превью меняется сразу);
- скрыть панель;
- скопировать коды цветов;
- отправить выбор в Telegram (после настройки бота в `demo-config.js`).

Ссылка с выбранной палитрой имеет вид `index.html?p=...` — её можно открыть,
чтобы увидеть сайт в этих цветах.

## Деплой на бесплатный хостинг

Сайт статический — никакой сборки не нужно.

**Netlify / Vercel:** перетащите папку проекта в их веб-интерфейс (drag&drop) —
готово.

**GitHub Pages:** залейте папку в репозиторий, в настройках включите Pages
(ветка `main`, корень `/`).

> Важно: имена файлов фото в верхнем регистре (`.JPG`). На хостинге регистр
> учитывается — не переименовывайте, либо обновите пути в `config.js`.

## Совет по скорости

Фото по 600–900 КБ. Перед публикацией желательно сжать их (например, squoosh.app
или tinypng.com) до ~150–250 КБ — сайт будет грузиться заметно быстрее.
